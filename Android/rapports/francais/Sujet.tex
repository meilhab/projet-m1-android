\chapter{Présentation du sujet}


\section{Le sujet}

La plate-forme Android permet au développeur d'avoir énormément de possibilité dans la création d'application. C'est d'ailleurs pour cela qu'il y a tant d'applications disponibles sur l'Android Market.

Utiliser un maximum de fonctionnalités dans une application afin d'avoir une vue de ce qu'il est possible de faire est le but final du projet. Pour gagner du temps de développement, le logiciel réalisé au premier semestre de Master 
(un gestionnaire de tâche) dans le cadre de l'unité de valeur \textit{Modélisation, Interface utilsateur, Conception Avancée} a été choisie. 

\noindent De ce fait, le projet consiste essentiellement à :
\begin{itemize}
    \item adapter l'application existante à un smartphone\label{smartphone} Android;
    \item correctement ajouter les différentes fonctionnalités utiles au smartphone;
    \item avoir une application finale fonctionnelle.

\end{itemize}



\section{Le gestionnaire de tâche}


\subsection{Présentation de l'existant}

Le gestionnaire de tâche réalisé dans le cadre de l'UV\protect\footnote{Unité de Valeur} MICA\protect\footnote{Modélisation, Interface utilsateur, Conception Avancée} est un logiciel de gestion des tâches quotidiennes que chacun doit effectuer. C'est un aide mémoire utilisable par tout le monde.

\subsubsection{Fonctionnalités existantes}

Ce logiciel initialement de type RCP\protect\footnote{Rich Client Platform} pour l'IDE\protect\footnote{Integrated Development Environment, Environnement de Développement Intégré} Eclipse permet de réaliser les actions suivantes :
\begin{itemize}
    \item gestion des tags\protect\footnote{marqueur lexical, notamment utilisé dans les réseaux sociaux, permettant la catégorisation d'une tâche autour d'un thème};
    \item gestion des tâches;
    \item trier les tâches selon certains critères;
    \item attribuer des sous-tâches à certaines tâches;
    \item changement de langue via une internationalisation du logiciel en anglais;
    \item sauvegarder la liste des tâches dans un format XML\protect\footnote{eXtensible Markup Language};

\end{itemize}

\subsection{Présentation du logiciel pour smartphone}

Reprenant les fonctionnalités du logiciel existant, le gestionnaire de tâche doit s'adapter à un smartphone, c'est-à-dire s'adapter à la taille de l'écran, posséder une interface simple et non surchargée tout en restant complet.
Il doit aussi inclure de nouvelles possibilités existantes avec Android.

\noindent En reprenant certains des points vus précédemment, le gestionnaire de tâche permet de :
\begin{itemize}
    \item gérer les tags (ajout/suppression);
    \item gérer les tâches (ajout/modification/suppression);
    \item gérer les sous-tâches des tâches mères;
    \item utiliser la base de donnée SQLite disponible sur le smartphone Android;
    \item se synchroniser avec un serveur distant;
    \item utiliser diverses fonctionnalités inclues avec Android;
    \item gestion de comptes utilisateurs sur le serveur.

\end{itemize}
